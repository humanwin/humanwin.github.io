MySQL基本操作命令

安装

  1. 下载mysql-8.0.25-winx64.zip并解压
  2. 进入mysql-8.0.25-winx64\bin目录下，命令行执行mysqld --initialize --console
  3. 执行完会看到为root@localhost账号随机生成了一串密码XXX，先保存下来.

启动MySQL

命令行执行net start mysql,提示MySQL 服务已经启动成功。

登陆MySQL

命令行输入mysql -u root -p回车，然后输入上面随机生成的密码XXX，命令行显示mysql> 说明登陆成功

创建用户

create user '用户名'@'host' identified by '密码';

  1. 磁盘上，往mysql.user表里插入一行，由于没有制定权限，这行数据上所有表示权限的字段的值都是N.
  2. 内存上，往数组acl_users里插入了一个acl_user对象，这个对象的access字段值为0.

用户权限

  1. 授予全局权限，作用于整个MySQL实例，权限信息保存在mysql.user表里。不影响已有连接。
1

grant all privileges on *.* to '用户名'@'host' with grant option;
  1. 授予db权限，作用于整个MySQL实例，权限信息保存在mysql.db表里。不影响已经拿到这个库的权限的连接。
1

grant all privileges on *db1.* to '用户名'@'host' with grant option;
  1. 授予列权限和表权限，马上影响到已经存在的连接。权限信息保存在mysql.tables_pri表和mysql.columns_priv表里。
1
2
3

create table db1.t1(id int,a int);
grant all privileges on db1.t1 to '用户名'@'host' with grant option;
grant select(id),insert(id,a) on mydb.mytal to '用户名'@'host' with grant option;
  1. 取消权限
1

revoke all privileges on *.* from '用户名'@'host';
  1. 查看权限
1

show grants for 用户名;
  1. flush privileges：正常使用grant命令不需要执行。当直接用DML语句操作系统权限表后，表中权限更内存中权限不一致时，可使用该命令重建内存达到一致。

查看数据库相关信息

  1. 端口
1

show variables like 'port';
  1. 数据库大小
1
2
3
4
5
6
7

use information_schema;
select concat(
    round(sum(data_length)/(1024*1024),2)+
    round(sum(index_length)/(1024*1024),2),
    'MB') 
    as 'DB Size' from tables 
    where table_schema='mysql';

备份和恢复

  1. 备份
1
2

mysqldump.exe -u username -ppassword --databases 库1 [库2 ... 库n] > 库备份文件名.sql
mysqldump.exe -u username -ppassword -t 库名 --tables 表1 [表2 ... 表n] > 表备份文件名.sql
  1. 删除数据库(可以认为是意外损坏了)
1

drop database 库名;
  1. 恢复
1
2
3
4

create database 库名;//先创建出需要恢复的数据库
mysql.exe -u username -ppassword  库名 < 库备份文件名.sql //整个库导入
create table 表定义;//先创建出需要恢复的表
mysql.exe -u username -ppassword  库名 < 表备份文件名.sql //整个表导入
